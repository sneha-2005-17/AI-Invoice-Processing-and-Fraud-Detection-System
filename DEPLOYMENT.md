# Deployment Guide

## Local Docker

```bash
docker compose up --build
```

Services:

- Frontend: `http://localhost:3001`
- Backend: `http://localhost:8000`
- API docs: `http://localhost:8000/docs`
- PostgreSQL: `localhost:5432`
- ChromaDB: `localhost:8001`

## Render Backend

1. Create a PostgreSQL database on Render or another managed provider.
2. Create a web service from `backend/render.yaml`.
3. Set `DATABASE_URL`, `GEMINI_API_KEY`, and `ALLOWED_ORIGINS`.
4. Keep `SECRET_KEY` generated by Render or set a strong secret manually.

## Vercel Frontend

1. Import the `frontend/` directory.
2. Set `NEXT_PUBLIC_API_URL` to the deployed Render backend URL.
3. Deploy with the default Next.js framework preset.

## Production Notes

- Replace local ChromaDB with a persistent production vector store.
- Store uploads and reports in object storage for multi-instance deployments.
- Add Alembic migrations before long-lived production schema evolution.
- Rotate API keys and use provider-specific secret managers.
- Configure monitoring for API latency, OCR latency, Gemini spend, and fraud review outcomes.
